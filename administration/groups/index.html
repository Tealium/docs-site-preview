Permission groups

This article describes how to manage users on your account in the platform permissions system.

How it works

A permission group defines a set of access levels for the profiles in your account, product features, and publishing rights. Users are assigned to one or more groups, and have the permissions and access associated with the groups.

Users in multiple permission groups

Access rights to profiles is additive. So, if multiple permission groups are allowed access to the same profile and a user is a member of those permission groups, the user will use the permission group with the greatest possible access.

For example, Group A has View permissions on a profile, while Group B has Edit and View permissions on that profile. A user in both Group A and Group B will have Edit and View permissions on that profile.

Also, a user’s admin roles may affect profile permissions. A user with the Profile Admin role will have full permission on all profiles in the account.

Product features

Product and product feature permissions specify the Tealium products and features that users can access. You can configure a permission group to set permissions on the following products:

  • iQ Tag Management
  • AudienceStream
  • EventStream
  • DataAccess
  • DataConnect
  • Predict
  • Functions
  • Server-Side Tools
  • Server-Side (Others)

Product access permissions are divided into feature permissions, and may vary depending on the product permissions assigned to the group.

Users that have View & Edit or View, Edit & Delete permission also have Save permission for their changes to a profile.

Permissions to features in multiple products

Carefully consider when you disable access to a feature or product. If you disable access to a feature for a user, that user will not be able to access the feature in any other features that depend on it. For example:

  • If you disable access to AudienceStream audiences for a user, that user will not be able to select an audience to action in an AudienceStream connector.
  • If you disable access to server-side labels for a user, the user will not see labels on the server-side. They will still be able to access connectors or other elements with labels on them, but they will not see or be able to edit the labels themselves.

Publishing changes to features in multiple products

If you disable a product for a user, but the user has server-side Publish permission and access to another product that shares elements with the first product, the user can still publish changes to those shared elements. For example, EventStream and AudienceStream share some elements, such as rules and labels.

If a user has Publish permission and only has access to EventStream, they can publish changes to rules and labels they made in EventStream, and those changes will also affect AudienceStream. Similarly, if a user has Publish permission and only has access to AudienceStream, their published changes to rules and labels also affect EventStream.

Default permission groups

The system creates the following default groups:

  • Viewers - Viewing permissions on all products and features on the account.
  • Editors - Edit permissions on all products and features on the account.
  • Publishers - Publish permissions for versions on client-side and server-side.

These groups do not contain any users or be assigned to any profiles. You can modify each of these groups as needed (name change, add or remove members, delete the group, et cetera).

Manage groups

View groups

From the user menu, click Manage Permissions to access the Manage Permissions screen. The Groups table will appear.

View a group

Click a permission group to display a dialog with that group’s details. You can click on a tab to display more details about that group:

  • Users - Lists the users who are a member of the permissions group.
  • Features - Lists each product’s features and the permissions the group’s members has on those features.
  • Profiles - Lists the profiles and the permissions the group’s members has on those profiles.

Create a group

To create or edit a permission group, you must have the Account Admin or the User Admin role. To add a permission group to a profile, you must have the Account Admin or the Profile Admin role.

When you create a permission group, you will set the access level for each product and its features, and then invite new users to join the account and the permission group.

To create a new permission group:

  1. From the user menu, click Manage Permissions. The Manage Permissions screen will appear.
  2. Click + New Group. The New Permission Group dialog appears.
  3. In the Name box, enter a name for the group.
  4. Select what the new group will initially contain:
    • Blank Group - Start a new group without any existing settings.
    • Duplicate an Existing Group - Start a new group with the same permissions, profiles, and users of the group that you select.
  5. Click Next. The Set Account Permissions dialog appears.
  6. If you want to give the group permissions to publish profile changes, under Server-Side select Publish.
  7. Select the products you want the group to access.
  8. Click Next. The Feature Permissions dialog appears.
  9. Under Feature Permissions, select the access level for each product.
  10. If you want to set permissions for each of a product’s features, select Custom and then set the permissions for the individual features.
    • Some features appear in multiple products. Feature access levels apply to the feature, regardless of which product it appears in.
    • Edit access for a feature or product includes the ability to save changes to the profile.
  11. Click Next. The Profiles dialog appears.
  12. Select the profiles to allow the group to access.
  13. Click Next. The Add Users dialog appears.
  14. Select up to 25 user email addresses to add to the permission group.
    • To remove an email address from the list, click the delete icon in the user’s row.
    • To add a user who is not already on your account, see Managing Users.
  15. Click Add.
    • If you do not add any users to the group, a warning dialog will appear. If you want to add users later, click Continue. Otherwise, click Cancel and add users to the group.

The Manage Permissions screen will appear, and the new group will appear in the list.

Edit a group

To edit a permission group:

  1. From the user menu, click Manage Permissions. The Manage Permissions screen will appear.
  2. Click the Permission Group tab.
  3. Click the permission group you want to edit. The group’s details will appear.
  4. Select the information you want to edit:
    • Group Permissions - You can add or remove product features from the permission group, and you can set the permission group’s access levels for product features.
    • Users - You can add or remove users from the permission group.
    • Profiles - You can add or remove profiles from the permission group, and you can set the permission group’s access levels for that profile.

Manage a user’s groups

To add or remove a user from that user’s profile:

  1. From the user menu, click Manage Permissions. The Manage Permissions screen will appear.
  2. Click the User tab.
  3. Click the user that you want to edit. The User Details dialog will appear.
  4. Click the Group tab, and then click Edit.
  5. Select the permission groups you want to add the user to or remove them from.
  6. Click Finish.
