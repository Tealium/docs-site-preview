SWIFT

クラス：TealiumConfig

メインのTealiumクラスの構成オプションを設定するためのクラスです。 個々のモジュールでは、TealiumConfigクラスにモジュール独自の拡張機能を提供します（有効な場合）。

サポートされているプラットフォーム

  • iOS
  • tvOS
  • watchOS
  • macOS

インストール

TealiumCoreフレームワークに同梱されています。

CocoaPods

pod 'tealium-swift/TealiumCore'

import TealiumSwift

Carthage

TealiumCore.framework

import TealiumCore

パブリックAPI

TealiumConfig()

TealiumConfigオブジェクトのコンストラクタ。 このオブジェクトは、特定のアカウントのライブラリを初期化するために必要です。

TealiumConfig(account:String, profile:String, environment:String, optionalData:Dictionary, dataSource:String)
パラメータ 型 詳細
account 文字列 Tealiumアカウントの名前
profile 文字列 Tealiumプロファイルの名前（通常は”main”）
environment 文字列 Tealiumの環境：”dev”\、”qa”\、または”prod”
optionalData [String: Any]? パラメータは任意であり、不要な場合は省略できます。 optionalDataの説明は以下を参照してください。
datasource 文字列 Datasourceキー（UDHから取得）、 任意。 
// standard basic config object
let tealConfig = TealiumConfig(account: "your_account", profile: "your_profile", environment: "dev/qa/prod")

// with optionaldata and datasource
let tealConfig = TealiumConfig(account: "your_account", profile: "your_profile", environment: "dev/qa/prod", optionalData: nil, datasource: "abc12")

// with datasource but no optionalData
let tealConfig = TealiumConfig(account: "your_account", profile: "your_profile", environment: "dev/qa/prod", datasource: "abc12")

optionalData

初期化時に特定のデフォルト動作のオーバーライドを許可します。 一般的には、optionalDataオブジェクトを明示的に指定するのではなく、オーバーライドメソッドの使用が推奨されます。

OptionalDataキー データ型 機能
TealiumTagManagementConfigKey.maxQueueSize 整数 キャッシュされた（オフライン）イベント用のキューの最大サイズを表す整数
TealiumTagManagementConfigKey.overrideURL 文字列 Tag Management WebviewのオーバーライドURL
TealiumCollectKey.overrideCollectUrl 文字列 Tealium Collect HTTPSリクエストのオーバーライドURL
TealiumDeviceDataModuleKey.isMemoryReportingEnabled ブーリアン 低レベルデバイスのメモリレポートをオン（true）またはオフ（false、デフォルト）にする
TealiumAttributionKey.isSearchAdsEnabled ブーリアン 属性モジュールでApple Search Ads APIのサポートを有効にする（デフォルトはオフ（false））

setModulesList()

有効または無効にするモジュールのリスト （モジュールのリストを参照してください）。

setModulesList(TealiumModulesList)
パラメータ 型 詳細
modulesList TealiumModulesList* 有効または無効にするモジュールのリスト 
// blacklists the autotracking module
let modulesList = TealiumModulesList(isWhitelist: false, moduleNames: ["autotracking"]
config.setModulesList(modulesList)

*TealiumModulesList構造体

パラメータ 型 詳細
isWhiteList ブーリアン モジュールのリストがブラックリスト（false）とホワイトリスト（true）のどちらであるかを判別
modulesList [文字列] 有効または無効にするモジュール名のarray

getModulesList()

現在のモジュールリストオブジェクトを返します （モジュールのリストを参照してください）。

getModulesList() -> TealiumModulesList

// blacklists the autotracking module
let modulesList = config.getModulesList()

setLogLevel()

新しいログレベルを設定します

setLogLevel(logLevel: TealiumLogLevel)

パラメータ 型 詳細
logLevel TealiumLogLevel 新しいログレベル（SwiftのLoggerモジュールに関するページを参照） 
// .none .errors .warnings .verbose
let newLogLevel = TealiumLogLevel.errors
config.setLogLevel(logLevel: newLogLevel)

getLogLevel()

現在のログレベルを返します（SwiftのLoggerモジュールに関するページを参照）

getLogLevel() -> TealiumLogLevel

let logLevel = config.getLogLevel()

setTagManagementOverrideURL()

Tag Managementモジュールで使用されるデフォルトのURLをオーバーライドします。 これは、Tealium JavaScriptファイルをセルフホストしている場合に必須です。

setTagManagementOverrideURL(string: String)

パラメータ 型 詳細
string 文字列 読み込むURLを表す文字列 
let url = "https://tags.mycdn.com/utag/myaccount/myprofile/myenv/mobile.html"
config.setTagManagementOverrideURL(string: url)

setTagManagementQueueSize()

タグ管理キューの最大サイズを設定します

setTagManagementQueueSize(to: Int)

パラメータ 型 詳細
to 整数 キューの最大サイズ 
config.setTagManagementQueueSize(to: 1)

setRootView()

WKWebViewがアタッチされる現在のUIViewを設定します。

setRootView(UIView)

パラメータ 型 詳細
view UIView WKWebViewがアタッチされるUIViewインスタンス 
config.setRootView(self.view)

setShouldUseLegacyWebview()

WKWebViewではなくUIWebViewの使用を強制します。

setShouldUseLegacyWebview(shouldUse: Bool)

パラメータ 型 詳細
shouldUse ブーリアン WKWebViewではなくUIWebViewを使用する場合はtrue 
config.setShouldUseLegacyWebview(shouldUse: true)

setCollectOverrideURL()

データを別のエンドポイントに送信するようにTealium CollectのURLをオーバーライドします。

setCollectOverrideURL(string:String)

パラメータ 型 詳細
string 文字列 Collectモジュールからのデータの送信先となるURLを表す文字列。 データ収集を特定の地域（例：中央ヨーロッパ、ドイツ）内に限定するために使用できます。 
// force override tealium collect base URL to EU Central
let url = "https://collect-eu-central-1.tealiumiq.com/vdata/i.gif?"
config.setCollectOverrideURL(string: url)

setMemoryReportingEnabled()

DeviceDataモジュールでメモリレポートを有効または無効にします（デフォルトでは無効）。

setMemoryReportingEnabled(Bool)

パラメータ 型 詳細
enabled ブーリアン DeviceDataモジュールでメモリレポートを有効化または無効化します。 
config.setMemoryReportingEnabled(true)

isMemoryReportingEnabled()

メモリレポートが現在有効になっているか無効になっているかを示すブーリアンを返します。

isMemoryReportingEnabled() -> Bool

let enabled = config.isMemoryReportingEnabled()

setSearchAdsEnabled()

DeviceDataモジュールでメモリレポートを有効または無効にします（デフォルトでは無効）。

setSearchAdsEnabled(Bool)

パラメータ 型 詳細
enabled ブーリアン 属性モジュールでApple Search Ads APIを有効または無効にします 
config.setSearchAdsEnabled(true)

isSearchAdsEnabled()

属性モジュールで現在Apple Search Ads APIが有効になっているか無効になっているかを示すブーリアンを返します。

isSearchAdsEnabled() -> Bool

let enabled = config.isSearchAdsEnabled()

disableRemoteHTTPCommand()

ビルトインリモートHTTPコマンドを無効にします（SwiftのRemoteCommandsモジュールに関するページを参照）。

disableRemoteHTTPCommand()

config.disableRemoteHTTPCommand()

enableRemoteHTTPCommand()

ビルトインリモートHTTPコマンドを有効にします（SwiftのRemoteCommandsモジュールに関するページを参照）。

enableRemoteHTTPCommand()

config.enableRemoteHTTPCommand()

dispatchQueue()

すべてのTealiumプロセスに使用されている現在のディスパッチキューを返します。

ispatchQueue() -> DispatchQueue

let dispatchQueue = config.dispatchQueue()

setDispatchQueue()

Tealiumで使用する新しいディスパッチキューを設定します （非推奨）。 デフォルトはDispatchQueue.mainです。

setDispatchQueue(DispatchQueue)

パラメータ 型 詳細
DispatchQueue DispatchQueue Tealiumで使用するディスパッチキュー 
let dispatchQ = DispatchQueue(label: "com.tealium.myqueue")
config.setDispatchQueue(dispatchQ)

defaultDispatchQueue()

デフォルトのディスパッチキュー（DispatchQueue.main）を返します。

defaultDispatchQueue() -> DispatchQueue

let dispatchQueue = config.dispatchQueue()

addDelegate()

TealiumDelegateプロトコルに準拠する新しいデリゲートを追加します。

addDelegate(Delegate)

パラメータ 型 詳細
delegate TealiumDelegate TealiumDelegateプロトコルに準拠するクラス 
// assuming current module has implemented the TealiumDelegate protocol
config.addDelegate(self)

delegates()

現在有効なデリゲートのarrayを返します。

delegates() -> [TealiumDelegate]

let delegates = config.delegates()

setConnectivityRefreshEnabled()

有効な場合（デフォルト）、Connectivityモジュールは、（setConnectivityRefreshIntervalで制御された）指定された間隔で接続を確認します。 その後、キューに追加されたディスパッチは、接続が再開されるとすぐに送信されます。

setConnectivityRefreshEnabled(Bool)

パラメータ 型 詳細
enabled ブーリアン Connectivityモジュールで接続更新機能のenabled状態を設定します。 デフォルトはtrue。 
config.setConnectivityRefreshEnabled(true) // set to false to disable this feature

setConnectivityRefreshInterval()

デフォルトの接続更新間隔を秒単位で設定します。 TealiumConfigインスタンスで設定されていない場合、デフォルトは30秒です。

setConnectivityRefreshInterval(Int)

パラメータ 型 詳細
interval 整数 デバイスがオフラインの場合にConnectivityモジュールが接続のテストを行うまでの秒数 
config. setConnectivityRefreshInterval(5) // sets interval to 5 seconds

addRemoteCommand()

後で実行するためのリモートコマンドを追加します

addRemoteCommand(TealiumRemoteCommand)

パラメータ 型 詳細
command TealiumRemoteCommand 追加するTealiumRemoteCommandのインスタンス 
// run prior to Tealium initialization
#if os(iOS)
let remoteCommand = TealiumRemoteCommand(commandId: "test",
                                         description: "test") { response in
			print("Remote Command 'test' executed")
}
config.addRemoteCommand(remoteCommand)
#endif

getRemoteCommands()

TealiumConfigインスタンスに設定されているリモートコマンドのarrayを返します

getRemoteCommands() ->[TealiumRemoteCommand]?

let remoteCommands = config.getRemoteCommands()

setLegacyDispatchMethod()

デフォルトの”/event” POSTリクエストディスパッチャーではなく、従来の”vdata” GETリクエストディスパッチャーの使用をCollectモジュールに強制します。

setLegacyDispatchMethod(Bool)

パラメータ 型 詳細
_ ブーリアン trueの場合は、従来の”vdata”ディスパッチメソッドを使用するようCollectモジュールに強制します 
config.setLegacyDispatchMethod(true)
