Tag Managementモジュール

Tag Managementモジュールは、クライアントサイドでのユニバーサルタグ（utag.js）の実装で、レンダリングされないWKWebViewインスタンスを使用してJavaScriptを実行します。

使用方法

このモジュールは、iQ Tag Managementを利用するアプリで必要です。 これはCarthageおよびCocoaPodsフレームワークビルドに自動的に含まれています。

サポートされているプラットフォーム

  • iOS

インストール

CocoaPods

ポッドがインストールされている場合、フレームワークは自動でインスタンス化されます。 TealiumCoreポッドに依存します。 import TealiumSwift以外には、追加のimportステートメントは必要ありません。

pod 'tealium-swift/TealiumTagManagement'

Carthage

フレームワークがアプリケーションにコンパイルされている場合（”埋め込みバイナリ”に含まれている場合）、フレームワークは自動でインスタンス化されます。 TealiumCoreに依存します。 追加のimportステートメントは必要ありません。

TealiumTagManagement.framework

外部の依存関係

  • UIKit（WKWebView）

含まれる変数

次の変数は、トラッキングコールのたびに送信されます。

変数 詳細 値の例
dispatch_service トラッキングコールの発生元であるモジュールを示す静的文字列 tagmanagement

パブリックAPI

WKWebViewプロトコルは、WKWebViewデリゲートに準拠する別のオブジェクトをモジュールのマルチキャストデリゲートに追加することでアクセス可能になります。

tealium.tagManagement()?.delegates.add(MY_OBJECT)

WKWebViewについて

バージョン1.7.0より前でこのモジュールに使用されていたUIWebViewは、現在、WKWebViewを優先するために非推奨になりました。 このOSでは、レンダリングされない場合でも、WKWebViewが常にUIViewにアタッチされている必要があります。 UIViewにアタッチされていない場合は、OSのパフォーマンスが大幅に低下するため、JavaScriptは準最適に実行されます。 このためには、Tag Managementモジュールでは、レンダリングされないWKWebViewをアプリのルートUIViewにアタッチしようとします。

アプリのビュー階層が複雑で、モジュールでrootViewControllerを検出できない場合は、viewDidLoad、viewWillAppear:、またはviewDidAppear:でアタッチするビューを使用してTealiumConfigを初期化することができます。

リリース1.7.1では、ビューの自動検出機能が大幅に改善され、ほとんどのシナリオでビューを指定する必要がなくなりました。 ただし、トラッキングコールが正しく送信されたことを確認するには、これをアプリでテストする必要があります。 詳細については、リリースノートを参照してください。

プッシュ通知の受信時にアプリでUIViewControllerに移動して、表示されるビュー階層がTealiumの初期化時と異なる場合は、次のAPIメソッドを使用して現在のUIViewを更新する必要があります。

override func viewDidLoad() {
    super.viewDidLoad()
    // this view does not have a navigation controller
    if self.navigationController == nil {
          // update the current root UIView on the Tealium instance
        tealium?.updateRootView(self.view)
    }
    // Do any additional setup after loading the view.
}

完全な例については、Swift-WKWebViewサンプルアプリを参照してださい。

WKWebViewとUIWebViewはそれぞれ、異なるメソッド（WKHTTPCookieStoreとHTTPCookieStorage）を使用してCookieを格納するため、このモジュールでは、WKWebViewの初回起動時に自動的にCookieが同期されます。 HTTPCookieStorage内のCookieは削除されませんが、最終的には有効期限切れになります。

WKWebViewで問題が発生していて、UIWebViewの使用が好ましい場合は、デフォルトをオーバーライドするためのオプションについてTealiumConfigに関するページを参照してください。
