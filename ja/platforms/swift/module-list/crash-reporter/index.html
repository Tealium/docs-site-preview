CrashReporterモジュール

アプリのクラッシュを自動的にトラッキングします。

目的

Tealium Crash Reporterでは、アプリのクラッシュを自動的にトラッキングします。 このモジュールが有効になり、付属のフレームワークがインストールされると、アプリでクラッシュが発生した場合にクラッシュイベントがトリガーされます。

サポートされているプラットフォーム

  • iOS

インストール

アプリにTealiumCrashReporteriOS.frameworkを追加する方法は3つあります。

CocoaPods

ポッドがインストールされている場合、フレームワークは自動でインスタンス化されます。 TealiumCoreおよびTealiumCrashReporteriOSポッドに依存します。 追加のimportステートメントは必要ありません。

  1. Podfileで次のエントリを使用します（Swift CocoaPodsのインストールに関するセクションを参照）： pod tealium-swift/Crash
  2. ターミナルウィンドウでpod installを実行します。
    • これにより、外部TealiumCrashReporteriOS依存関係がTealiumAppData、TealiumDeviceData、Coreの各モジュールと共に自動的にインストールされます。

Carthage

フレームワークがアプリケーションにコンパイルされている場合（”埋め込みバイナリ”に含まれている場合）、フレームワークは自動でインスタンス化されます。 TealiumCoreおよびTealiumCrashReporteriOSに依存します（外部の依存関係）。

  1. Swift Carthageのインストールに関するセクションで説明している手順に従います。
  2. ターミナルウィンドウでcarthage update --platform iOSを実行します。
    • これにより、以下が生成されます。
      • TealiumCore.framework + すべての標準Tealiumフレームワーク
      • TealiumCrash.framework
      • TealiumCrashReporteriOS.framework
    • これらのフレームワークをすべてアプリ内に含めます。
  3. 関連するBuild Phasesスクリプトを追加する際は、App Storeへのアプリ提出時に問題が起きないように、Carthageを初めて使う方のためのドキュメントに記載された手順に従ってください。

手動

  1. Swiftの手動インストールに関するセクションで説明している手順に従います。
  2. GitHubレポジトリからTealiumCrashReporteriOS.frameworkをダウンロードします。
  3. TealiumおよびTealiumCrashスキームをビルドします。
  4. TealiumCore.framework、TealiumCrash.framework、TealiumAppData.framework、TealiumDeviceData.framework、TealiumCrashReporteriOS.frameworkをXcodeの[Embedded Binaries]セクションにドラッグアンドドロップします。

外部の依存関係

TealiumCrashReporteriOS

推奨される使用方法

アプリのクラッシュに関する重要な情報を取得するために、このフレームワークの使用を推奨します。

含まれる変数

モジュールが有効な場合に、トラッキングコールのたびに以下の変数が送信されます。

変数名 詳細 値の例
crash_uuid この特定のクラッシュの一意識別子 CC2DA0E9-E544-429A-AC5E-A268FC62F02A
device_memory_usage* クラッシュ時にアプリによって使用されているデバイスの現在のメモリ 143.57MB
app_memory_usage クラッシュ時にアプリによって使用されているデバイスの現在のメモリ 143.57MB
device_memory_available* クラッシュ時のデバイスの空きメモリ 1068.88MB
memory_free クラッシュ時のデバイスの空きメモリ 1068.88MB
device_os_build 現在のOSのビルド番号 15E217
crash_process_id クラッシュ時のアプリのPID 84351
crash_process_path アプリが実行されていたパス /DemoApp.app/DemoApp
crash_parent_process アプリを起動した親プロセス launchd_sim
crash_parent_process_id 親プロセスのPID 83183
crash_name わかりやすいクラッシュの名前（該当する場合） クラッシュ名
crash_cause クラッシュの原因 クラッシュの理由
crash_signal_code クラッシュの原因となっている信号コード #0
crash_signal_name クラッシュの原因となっている信号名 SIGABRT
crash_signal_address クラッシュの信号アドレス 4572945982
crash_libraries クラッシュ時の読み込み済みのライブラリのリスト [{“baseAddress”: “0xa3e0000”, “codeType”: { “arch”: 64, “typeEncoding”: “Mach”}, “imageName”: “/Applications/Xcode9.3.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/Library/CoreSimulator/Profiles/Runtimes/iOS.simruntime/Contents/Resources/RuntimeRoot/usr/lib/dyld_sim”, “imageSize”: 212992, “imageUuid”: “4015e9b70bde”}]
crash_threads クラッシュ時のアクティブなスレッドをすべて返します。 {“crashed”: 1,“registers”: {“cs”: “0x07”},“stack”: {“instructionPointer”: 4572945982,“symbolInfo”: {“symbolName”: “”,“symbolStartAddr”: 0},“threadId”: “”}}

* この変数は、以前のリリースとの後方互換性のために重複しています。

初期化

TealiumCrashReporteriOS.frameworkをプロジェクトに追加したら、それ以外に必要な作業はありません。 クラッシュモジュールが有効になっていて、Tealium SDKがインストールされている限り、アプリでクラッシュが発生すると、クラッシュデータが自動的にトラッキングコールに追加されます。

パブリックAPI

// import area
// For Cocoapods
import TealiumSwift

// For Carthage or Manual installation
import TealiumCrash

// Invoke a crash
TealiumCrashReporter.invokeCrash(name: "TEST_CRASH_NAME", reason: "TEST_CRASH_REASON")
