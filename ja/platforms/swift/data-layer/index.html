データレイヤー

各モジュールによって提供されるデータレイヤー変数の全一覧です。

AppDataモジュールの変数

以下の変数は、AppDataモジュールによってデータレイヤーに追加されます。

変数名 詳細
app_build アプリのマイナービルドバージョン

例：2213
app_name アプリの名前（通常はアプリストアの名前と同じ）

例：Digital Velocity
app_rdns アプリバンドルの完全修飾名

例：com.tealium.digitalvelocity
app_uuid ランダムなUUID。 いずれかの永続的ストレージモジュールも有効な場合、アプリをインストールしている間は維持されます。 アプリがアンインストールされるとリセットされます。

例：123e4567-e89b-12d3-a456-426655440000
app_version アプリバンドルのバージョン

例：1.0
app_version_full アプリのバージョンとビルド番号

例：1.0 (2213)
tealium_vid 永続的なTealium訪問者ID

例：123e4567e89b12d3a456426655440000
tealium_visitor_id tealium_vidと同じ （レガシー上の理由で含まれる）

例：123e4567e89b12d3a456426655440000

Attributionモジュールの変数

以下の変数は、Attributionモジュールによってデータレイヤーに追加されます。

変数名 詳細
ad_campaign_id 対応する広告が一部となっていたキャンペーンのID

例：1234567890
ad_campaign_name 対応する広告が一部となっていたキャンペーンの名前

例：CampaignName
ad_group_id 対応する広告が一部となっていた広告グループのID

例：1234567890
ad_group_name 対応する広告が一部となっていた広告グループの名前。

例：AdGroupName
ad_keyword 対応する広告のクリックを導いた広告インプレッションを促したキーワード。

例：Keyword
ad_org_name 対応する広告が一部となっていたキャンペーンを所有する組織。

例：OrgName
ad_user_clicked_last_30_days アプリをダウンロードする前の30日以内にユーザーがSearch Adsインプレッションをクリックした場合はTrue。

例：true
ad_user_date_clicked ユーザーが対応する広告をクリックした日時

例：2016-12-05T17:31:40Z
ad_user_date_converted ユーザーがアプリをダウンロードした日時

例：2016-12-05T17:31:40Z
device_advertising_id ユーザーによるリセットが可能なAdvertising Identifier（IDFA）

例：6D92078A-8246-4BA4-AE5B-76104861E7DC
device_advertising_enabled ユーザーが広告のトラッキングを許可しているかどうか示すブーリアン。 広告のトラッキングが有効な場合はtrueで、それ以外の場合はfalse。 注：falseの場合は、広告IDは0の文字列として表示されます

例：true
device_advertising_vendor_id 単一ベンダーの同じデバイス上にあるすべてのアプリで必ず同一になる一意のID（com.tealiumまたはcom.acmeなど、RDNSバンドルIDの最初の2つの部分が同じになるアプリ）。

例：6D92078A-8246-4BA4-AE5B-76104861E7DC

Crash Reporterモジュールの変数

以下の変数は、Crash Reporterモジュールによってデータレイヤーに追加されます。

変数名 詳細/例
crash_cause クラッシュの原因

例：クラッシュの理由
crash_name わかりやすいクラッシュの名前（該当する場合）

例：クラッシュの名前
crash_libraries クラッシュ時の読み込み済みのライブラリのリスト

例：
`[{“baseAddress”: “0xa3e0000”, “codeType”: { “arch”: 64, “typeEncoding”: “Mach”}, “imageName”: “/Applications/Xcode9.3.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/Library/CoreSimulator/Profiles/Runtimes/iOS.simruntime/Contents/Resources/RuntimeRoot/usr/lib/dyld_sim”, “imageSize”: 212992, “imageUuid”: “4015e9b70bde”}]``
crash_process_id クラッシュ時のアプリのPID

例：84351
crash_process_path アプリが実行されていたパス

例：/DemoApp.app/DemoApp
crash_parent_process アプリを起動した親プロセス

例：launchd_sim
crash_parent_process_id

Example: Parent process PID		 83183
crash_signal_address クラッシュの信号アドレス

例：4572945982
crash_signal_code クラッシュの原因となっている信号コード

例：#0
crash_signal_name クラッシュの原因となっている信号名

例：SIGABRT
crash_threads クラッシュ時のアクティブなスレッドをすべて返します。

例：
`{“crashed”: 1,“registers”: {“cs”: “0x07”},“stack”: {“instructionPointer”: 4572945982,“symbolInfo”: {“symbolName”: “”,“symbolStartAddr”: 0},“threadId”: “”}}`
crash_uuid この特定のクラッシュの一意識別子

例：CC2DA0E9-E544-429A-AC5E-A268FC62F02A
device_memory_usage クラッシュ時にアプリによって使用されているデバイスの現在のメモリ

例：143.57MB
device_memory_available クラッシュ時のデバイスの空きメモリ

例：1068.88MB
device_os_build 現在のOSのビルド番号

例：15E217

Lifecycleモジュールの変数

以下の変数は、Lifecycleモジュールによってデータレイヤーに追加されます。

変数名 詳細
lifecycle_dayofweek_local コールが行われた現地曜日（1 =日曜日、2 =月曜日など）

例：4
lifecycle_dayssincelastwake 整数の増分で最後にwakeが検出されてから経過した日数

例：1
lifecycle_dayssincelaunch 整数の増分で最初のlaunchから経過した日数

例：23
lifecycle_dayssinceupdate 整数の増分でアプリのバージョン更新が最後に検出されてから経過した日数

例：46
lifecycle_diddetect_crash このlaunch/wakeイベント中にクラッシュが検出されたかどうか。 trueの場合のみ入力。

例：true
lifecycle_firstlaunchdate launch/wakeが最初に検出されたときのGMTタイムスタンプ（ISO8601形式、協定世界時）

例：2013-07-11T17:55:04Z
lifecycle_firstlaunchdate_MMDDYYYY MM/DD/YYYY形式のGMTタイムスタンプ

例：01/18/2012
lifecycle_hourofday_local コールが行われた現地時間（24時間表記）

例：true
lifecycle_isfirstlaunch コールが、最初のlaunch/wakeコールである場合にのみ存在

例：true
lifecycle_isfirstlaunchupdate コールが、更新検出後の最初のlaunch/wakeである場合にのみ存在

例：true
lifecycle_isfirstwakemonth コールが、月の最初のlaunch/wakeである場合にのみ存在

例：true
lifecycle_isfirstwaketoday コールが、1日の最初のlaunch/wakeである場合にのみ存在

例：true
lifecycle_launchcount 現バージョンのアプリのlaunchの総回数（最後の更新以降）

例：3
lifecycle_priorsecondsawake アプリが最後のlaunchからwake状態にあった期間の総秒数。 これまでのすべてのwakeを総計。 lifecycle_type: launchコールでのみ送信。

例：126
lifecycle_secondsawake アプリが最新のwake/launchからwake状態にあった期間の総秒数。

例：30
lifecycle_sleepcount アプリがsleepになった総回数（更新時にリセット）

例：5
lifecycle_totalcrashcount インストール後のクラッシュの総回数（アプリが削除された場合のみリセット）

例：21
lifecycle_totallaunchcount インストール後のlaunchの総回数（アプリが削除された場合のみリセット）

例：3
lifecycle_totalsecondsawake アプリがインストール後にwake/アクティブな状態にあった期間の総秒数（アプリが削除された場合のみリセット）

例：36
lifecycle_totalsleepcount アプリがインストール後にバックグラウンドに移行した総回数（アプリが削除された場合のみリセット）

例：”400”
lifecycle_totalwakecount インストール後のlaunchとwake両方の総回数（アプリが削除された場合のみリセット）

例：”563”
lifecycle_type ライフサイクルコールの種類

例：”launch”、”wake”、”sleep”
lifecycle_updatelaunchdate バージョン更新検出後、最初のwake/launchのGMTタイムスタンプ

例：2014-09-08T18:10:01Z
lifecycle_wakecount 現バージョンのアプリのlaunchとwake両方の総数（更新時にリセット）

例：”29”
