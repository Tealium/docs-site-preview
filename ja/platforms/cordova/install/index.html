CORDOVA

インストール

CordovaアプリケーションにTealiumプラグインをインストールします。

要件

  • Cordova 7.0.0
  • CocoaPods

このプラグインには以下のTealiumライブラリが含まれます。

  • Tealium for iOS (5.3.2)
  • Tealium for Android (5.4.2)

バージョン1.1.1以降、このプラグインにはフレームワークファイルが含まれていません。 代わりに、iOSとAndroid向けに個別の依存関係マネージャが使用されています。

サンプルアプリ

Cordovaレポジトリ内で提供されているサンプルアプリを1つ以上ダウンロードすることを推奨しています。 これは当社のライブラリ、トラッキングメソッド、ベストプラクティスの実装について把握するのに役立ちます。

  • Cordovaサンプルアプリ

インストール

NPM（推奨）

NPMのパッケージ名は以下のとおりです。

  • tealium-cordova-plugin

NPM経由でプラグインをインストールするには、プロジェクトディレクトリ内のターミナルから以下のコマンドを発行します。

cordova plugin add tealium-cordova-plugin

手動

NPMが使用できない場合は、プラグインを手動でインストールすることもできます。 コマンドラインインターフェイス（CLI）から、以下を実行してください。

cd your_project_folder/
cordova platform add <PLATFORM>
cordova plugin add </LOCAL_PATH_TO_TEALIUM_PLUGIN/>
cordova build <PLATFORM>

初期化

tealium.init({
    account                  : "YOUR_ACCOUNT"
    , profile                : "YOUR_PROFILE"
    , environment            : "qa"
    , datasource             : "abc123"
    , instance               : "INSTANCE"
    , isLifecycleEnabled     : "false"
    , isCrashReporterEnabled : "true"
    , logLevel               : tealium.logLevels.DEV
    , collectDispatchProfile : "DISPATCH_PROFILE"
    , collectDispatchURL     : "DISPATCH_URL"
});
パラメータ 詳細
account アカウント名。
profile プロファイル名。
environment “qa”または”prod”。
datasource （任意）UDHのデータソースキー。
instance 一意のインスタンス名（マルチインスタンスに対応）。
isLifecycleEnabled （任意）ライフサイクルトラッキングを制御するための、”true”または”false”の文字列値（デフォルトは”true”）
isCrashReporterEnabled （任意。Androidのみ。Crash Reporterプラグインが必要）クラッシュレポートトラッキングを制御するための、”true”または”false”の文字列値
logLevel （任意）環境とは異なるログレベルを設定（デフォルトは環境に合わせた設定）。
オプション：tealium.logLevels.DEV、tealium.logLevels.QA、tealium.logLevels.PROD
collectDispatchProfile Tealium Collectエンドポイントで使用するプロファイルを設定。
collectDispatchURL Tealium CollectエンドポイントのフルURLをオーバーライド。

Tealium Collectのオーバーライド

デフォルトでは、iOSおよびAndroidのコアライブラリは、Tealium Collectのデータをアカウントの”メイン”プロファイルに送信します。 Tealium Collectを別のエンドポイントへ向けるには、プロファイルまたはエンドポイントURLをオーバーライドします。

例：プロファイルのオーバーライド

tealium.init({
    account                : "your_account",
    profile                : "us-mobile",
    environment            : "prod",
    instance               : window.tealium_instance,
    collectDispatchProfile : "us-mobile"
});

例：エンドポイントURLのオーバーライド

tealium.init({
    account            : "your_account",
    profile            : "us-mobile",
    environment        : "prod",
    instance           : window.tealium_instance,
    collectDispatchURL : "https://collect.tealiumiq.com/vdata/i.gif?tealium_account=your_account&tealium_profile=us-mobile"
});

ビルドのヒント

アプリのビルドに関して問題が発生した場合は、以下のコマンドを使ってプラットフォームの削除と再追加をする必要があります。コマンド内の<platform>は”android”または”ios”です。

cordova platform rm <platform>
cordova platform add <platform>

これにより通常は、物理デバイス向けのビルド時に発生する、XCodeのコードサイニング関連の偽エラーも解決します。

ビルドの実行中にCocoaPodsに関する問題が発生した場合は、pod cache clean --all && pod repo updateコマンドを実行し、ローカルのCocoaPodsのキャッシュをクリアします。

アンインストール

Tealiumプラグインを削除するには、ターミナルで以下のコマンドを実行します。

cordova plugin rm tealium-cordova-plugin

追加のTealiumモジュールがインストールされている場合は、併せて削除します（tealium-cordova-crashreportertealium-cordova-adidentifiertealium-cordova-installreferrer）。

リリースノート

変更の完全なリストについては、https://github.com/Tealium/cordova-plugin/blob/master/CHANGELOG.mdを参照してください。

後方互換性

1.1.0のリリースにより、APIの挙動がいくつか変更されています。 アップグレード後に以下の項目をテストしてください。

  • 過去にaddVolatileおよびaddPersistentメソッドを使用していた場合は、前述のサポートされた種類のデータのみを保存していることを確認します。
  • iOSでのパフォーマンス向上のために、初期化メソッドを除いたコードの大部分をバックグラウンドスレッドで実行するようになりました。 問題があれば、GitHubの問題ページから報告してください。
  • cordova_lifecycle変数は、以前はすべてのライフサイクルイベントのたびに送信されていましたが、削除されました。 この変数がまだ必要な場合は、Tealium iQの拡張機能であるSet Data Valuesのif lifecycle_type is populatedというルールを使用して変数を再作成し、ライフサイクルイベントに追加してください。
  • iOSで、ユーザーがアプリをバックグラウンドに移動するか、アプリを完全に終了させようとしたときに、すぐにライフサイクルのスリープイベントがトラッキングされるようになりました。 これは、アプリの実行中にユーザーがホームボタンを2回押してタスクスイッチャーを起動すると、スリープイベントが生成されるということです。 ユーザーがこのアクションを中止してアプリに戻ると、対応するウェイクイベントが生成されます。 以前は、スリープイベントはユーザーが（ホームボタンを1回押すか、別のアプリに切り替えて）バックグラウンドのアプリを終了したときにだけ生成されていました。 この挙動は現在のAndroidでの挙動と同じです。 そのため、分析ツールに大量のスリープイベントが記録される可能性がありますが、その数は技術的により正確なものとなります。

アップグレードのご案内

リリース1.1.1のプロジェクトからプラグインを削除し、追加し直すことを強く推奨します。

バージョン1.1.1では、プラグインのパッケージ名がcom.tealium.cordova.v5からtealium-cordova-pluginに変更されています。

前のバージョンのプラグインを削除するために、以下のコマンドの実行が必要になる場合があります。

cordova plugin rm com.tealium.cordova.v5
