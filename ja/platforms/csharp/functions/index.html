関数の定義

Tealium for C#のクラスと関数のリストです。

Config()

オブジェクトのコンストラクタを構成します。

public Config (string account,
               string profile,
               string dev,
               string visitorId,
               string[] modules,
               string datasource = null,
               string overrideCollectUrl = null,
               Dictionary<string, object> optionalData = null,
               Method method = Method.POST)
パラメータ 詳細 値の例
account Tealiumアカウント tealium
profile Tealiumプロファイル mobile_division
environment Tealium環境。 現時点では任意 prod
visitorID 32文字の英数字の文字列から成る識別子。1ユーザー、1アプリインスタンス、1デバイスごとに一意 612040730c8c11e6b4cbacbc329f41b7
modules ライブラリにより初期化されるモジュール 「使用可能なモジュール」を参照
dataSourceKey UDHのデータソースキー abc123
overrideCollectUrl Tealium Collectサービスのカスタムエンドポイント -
optionalData モジュールが使用するディクショナリ。 Nullも可。 通常、ほとんどの設定で値は不要です -
method Collect Endpointへのデータの送信方法を決定する、任意のHTTPメソッド（POSTまたはGET）。 Method.POST、Method.GET

Tealium()

プライマリオブジェクトのコンストラクタ。

Tealium(Config config)
パラメータ 詳細 値の例
config Configインスタンスが必要 -

Enable()

最初の初期化で自動的に呼び出されます。 既にライブラリが無効になっている場合は、Tealiumインスタンスとすべての内部モジュールを再度有効にします。

tealium.Enable()

Disable()

ライブラリモジュールによるイベントの一時的な処理を無効にします。 内部クラスオブジェクトの初期化が解除される場合があります。

tealium.Disable()

Track()

Track(string eventName, Dictionary<string, object> customData, TrackCompletion completion)
パラメータ 詳細 値の例
eventName トラッキングされたイベントに必要な文字列識別子 “aButtonWasTapped”
customData トラックコールに含めるキーと値のデータの任意のディクショナリ -
completion コール完了後にトリガーされる任意の完了ブロック - 
Dictionary<string, object> data = new Dictionary()
{
    {"foo1", new string[]{"a","12"}},
        {"foo2", "bar2"},
        {"foo3", "bar3"}
};
tealium.Track("My Event",
        customData,
        (success, info, error) =>
        {
        Debug.WriteLine("Track action completion block triggered.");
        Debug.WriteLine("Success: " + success + " info: " + info + " error: " + error);
});

TrackCompletion

トラックコールの任意のコールバックには、次のパラメータが含まれます。

パラメータ 詳細 値の例
successful トラックコールを配信できたかどうかを示すブーリアン true
Dictionary info トラッキングとレスポンスのデータを含むディクショナリ -
error ディスパッチサービスで配信またはレスポンスの問題が発生した場合の任意のエラー -

JoinTrace()

提供されたIDを使用してトレースに参加します。 その後、LeaveTrace()メソッドが呼び出されるまではすべてのトラックコールにtealium_trace_idが存在します。

//Sample
tealium.JoinTrace("01234");
パラメータ 詳細 値の例
traceId 参加するトレースを識別するのに必要な文字列 “12345”

LeaveTrace()

参加しているトレースから退出します。 その後のイベントでは、tealium_trace_idが自動的に設定されることはなくなります。

//Sample
tealium.LeaveTrace();

KillTraceSession()

Tealiumに追加イベントを送信して、サーバーの訪問者セッションを終了します。 これは、訪問終了の構成をテストする場合に役立ちます。 また、LeaveTrace()を呼び出して、以前のトレースIDを使用してそれ以上イベントが送信されないようにします。

//Sample
tealium.KillTraceSession();

使用可能なモジュール

現在のビルドでモジュールを有効にするには、次の文字列値を使用できます。

config.Modules = new string[] { AppDataModule.Name,
                                CollectModule.Name,
                                LoggerModule.Name };
モジュール名 詳細
AppDataModule コールに自動的なデータポイントを追加します
CollectModule Tealium Collectエンドポイントに処理済みのイベントを配信します
LoggerModule デバッグの出力を提供します

各モジュールクラスの.Name定数を使用してください。
