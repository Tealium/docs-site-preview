ANDROID

データレイヤー

Androidライブラリから利用可能なデータレイヤー変数の全一覧です。

以下の変数は、モバイルデータレイヤーの一部としてTealium Androidライブラリによって値が入力されます。 これらの変数はライブラリから自動的に各トラッキングコールに含まれ、Universal Data Hubの構成で使用することが可能です。

これらの変数は、モバイルでの使用のためにプロファイルをアクティベートするとiQ Tag Managementアカウントに追加されます。

Tealium for Android 5.0.xには、Lifecycle Trackingモジュールの追加変数を使用できます。

変数名 Androidのバージョン 詳細
app_build 5.4+ アプリのビルドバージョン（例：”1”）
app_name 1.1+ アプリケーション名（マニフェスト属性の”android:label”より）
app_memory_usage 5.4+ アプリプロセスで使用中のメモリ（MB）（例：”57”）
app_rdns 3.0+ 逆引きDNSのアプリケーションID（ルートマニフェストの”package”属性より）
app_version 1.1+ アプリケーションのバージョン[任意の文字列]
call_type 2.0+ コールの種類[view/link]
carrier 1.1+ 現在のデバイスのキャリア[ATT、Verizonなど]
carrier_iso 1.1+ キャリアのISO国コード - ISO 3166-1
carrier_mcc 1.1+ モバイルの国コード - ITU-T Recommendation E.212
carrier_mnc 1.1+ モバイルネットワークコード
connection_type 1.1+ [wifi、cellularなど]
device 1.1+ [iphone、ipadなど]
device_android_runtime 5.4+ Android Runtimeバージョン（例：”2.1.0”）
device_available_external_storage 5.4+ デバイスで使用可能な外部ストレージ
device_available_system_storage 5.4+ デバイスで使用可能なストレージ（例：273772544）
device_architecture 3.0+ デバイスのビットアーキテクチャ[32、64など]
device_battery_percent 4.0+ デバイスのバッテリー残量のパーセント（整数）[20、50など]
device_cputype 3.0+ CPUの種類[arm6、arm7、arm7sなど]
device_ischarging 4.0+ デバイスが現在充電中かどうか[true、false]
device_language 3.0+ 2文字のISO-639-1言語設定識別子[en、jaなど]
device_orientation 5.0+ コール時のデバイスの向き[Portrait、Landscape Left、Landscape Right、Portrait UpsideDown、Face Up、Face Down、Unknown]
device_os_build 5.4+ オペレーティングシステムのビルドバージョン（例：”4499259”）
device_os_version 5.0+ デバイスのオペレーティングシステムのバージョン[7.0]
event_name 5.0+ vdataに必要な”mobile_link”
library_version 1.2+ アプリによってインストールされたTealiumライブラリのバージョン
lifecycle_dayofweeklocal* 1.2+ コールが行われた現地曜日（例：1 =日曜日、2 =月曜日など）
lifecycle_dayssincelastwake* 1.2+ 整数の増分で最後にwakeが検出されてから経過した日数
lifecycle_dayssincelaunch* 1.2+ 整数の増分で最初のlaunchから経過した日数
lifecycle_dayssinceupdate* 1.2+ 整数の増分でアプリのバージョン更新が最後に検出されてから経過した日数
lifecycle_diddetectcrash* 5.0.3+ クラッシュが特定された場合にlaunchイベントでtrueが入力されます。
lifecycle_firstlaunchdate* 1.2+ launch/wakeが最初に検出されたときのGMTタイムスタンプ（ISO8601形式、協定世界時）
（例：2013-07-11T17:55:04Z）
lifecycle_firstlaunchdate_MMDDYYYY* 1.2+ MM/DD/YYYY形式のGMTタイムスタンプ
（例：01/18/2012）
lifecycle_hourofday_local* 1.2+ コールが行われた現地時間（24時間表記）
lifecycle_isfirstlaunch* 1.2+ コールが、最初のlaunch/wakeコールである場合にのみ存在（値 = true）
lifecycle_isfirstlaunchupdate* 1.2+ コールが、更新検出後の最初のlaunch/wakeである場合にのみ存在
lifecycle_isfirstwakemonth* 1.2+ コールが、月の最初のlaunch/wakeである場合にのみ存在（値 = true）
lifecycle_isfirstwaketoday* 1.2+ コールが、1日の最初のlaunch/wakeである場合にのみ存在（値 = true）
lifecycle_lastlaunchdate* 1.2+ 最後に記録されたlaunchのGMTタイムスタンプ（ISO8601形式、協定世界時）
（例：2013-07-11T17:55:04Z）
lifecycle_lastsleepdate* 1.2+ 最後に記録されたsleepのGMTタイムスタンプ（ISO8601形式、協定世界時）
（例：2013-07-11T17:55:04Z）
lifecycle_lastwakedate* 1.2+ 最後に記録されたwakeのGMTタイムスタンプ（ISO8601形式、協定世界時）
（例：2013-07-11T17:55:04Z）
lifecycle_launchcount* 1.2+ 現バージョンのアプリのlaunchの総回数（アプリのバージョン更新時にリセット）
lifecycle_priorsecondsawake* 1.2+ アプリが最後のlaunchからwake状態にあった期間の総秒数。 これまでのすべてのwakeを総計。 lifecycle_type:launchコールでのみ送信
lifecycle_secondsawake* 1.2+ アプリが最後のwake/launchからwake状態にあった期間の総秒数。 lifecycle_type:sleepでのみ送信
lifecycle_sleepcount* 1.2+ アプリがsleepになった総回数（アプリのバージョン更新時にリセット）
lifecycle_totalcrashcount* 5.0.4+ クラッシュが検出された総回数。 アプリの再インストール時を除いてリセットしないでください。
lifecycle_totallaunchcount* 1.2+ インストール後のlaunchの総数。
アプリの再インストール時を除いてリセットしないでください。
lifecycle_totalsecondsawake* 1.2+ アプリがwake/アクティブな状態にあった期間の総秒数。
アプリの再インストール時を除いてリセットしないでください。
lifecycle_totalsleepcount* 1.2+ アプリがインストール後にバックグラウンドに移行した総回数。
アプリの再インストール時を除いてリセットしないでください。
lifecycle_totalwakecount* 1.2+ インストール後のlaunchとwake両方の総回数。
アプリの再インストール時を除いてリセットしないでください。
lifecycle_type* 1.2+ ライフサイクルコールの種類（launch、wake、sleep）
lifecycle_updatelaunchdate* 1.2+ バージョン更新検出後、最初のwake/launchのGMTタイムスタンプ（ISO8601形式、協定世界時）（例：2013-07-11T17:55:04Z）
lifecycle_wakecount* 1.2+ 現バージョンのアプリのlaunchとwake両方の総数（アプリのバージョン更新時にリセット）
link_id 1.0+ （イベントコールのみ）ボタンのタイトルやオブジェクトのTealium IDなど、一般的なイベントまたはリンクのタイトル[任意の文字列]
~orientation~ 2.0+ 非推奨 コール時のデバイスの向き[Portrait、Landscape Left、Landscape Right、Portrait UpsideDown、Face Up、Face Down、Unknown]
origin 3.0+ ウェブの実装とモバイルを区別するためにマッピングで使用できる定数[mobile]
os_version 1.1+ デバイスのオペレーティングシステムのバージョン[7.0]
page_type 5.0+ モバイルビュー/モバイルページの変更を、モバイルイベント/ウェブイベントと区別するためにマッピングで使用できる”mobile_view”定数（vdataに必要）
platform 1.0+ [iOS、Android、BB10など]
screen_title 1.0+ （ビューコールのみ）一般的なビューのタイトル[任意の文字列]
timestamp 1.1+ イベント発生のタイムスタンプ（秒単位）（例：2013-07-11T19:57:47Z）[ISO8601形式、協定世界時]
timestamp_local 2.0+ イベント発生の現地時間タイムスタンプ（秒単位）（例：2013-07-11T19:57:47）[ISO8601形式、オフセットなし]
timestamp_offset 2.0+ 現地タイムゾーンの協定世界時からのオフセット（時間単位）（例：-3）
timestamp_unix 1.1+ イベント発生時のGMT/UTC UNIXタイムスタンプ（例：1373498679）
timestamp_unix_milliseconds 5.5.1 イベント発生時のGMT/UTC UNIXタイムスタンプ（ミリ秒単位）（例：1373498679123）
uuid 1.2+ （Universally Unique Identifier）デバイス上のアプリのインストールごとに一意[任意の文字列]
visitor_id 5.0+ Tealiumによって生成されたvisitor_id
