ANDROID

インストール

Tealium for Android SDKのコードを取得し、インストールします。

要件

  • Android Studio
  • Android APIレベル19以上（KitKat/4.4）

iQ Tag Managementと併用する場合：

*iQのモバイルプロファイル（「モバイルプロファイルの作成」を参照してください）

Universal Data Hubと併用する場合：

  • Tealium Universal Data Hubアカウント

コードの取得

GitHubからTealium Androidライブラリをクローンします。

  • https://github.com/Tealium/tealium-android

ライブラリをダウンロードせずクローンすることで、今後のリリースに向けたアップデートがしやすくなります。

Mavenインストール

TealiumではMavenを使用して依存関係のインストールと管理を行います。

Tealium MavenのURLを、プロジェクトのルートにあるbuild.gradleファイルに追加します。

allprojects {
    repositories {
        jcenter()
        maven {
            url "https://maven.tealiumiq.com/android/releases/"
        }
    }
}

Tealiumの依存関係をプロジェクトモジュールのbuild.gradleファイルに追加します。

dependencies {
    implementation 'com.tealium:library:5.5.5'
}

手動インストール

依存関係を手動で追加する手順は以下のとおりです。

  1. flatDirをプロジェクトのルートにあるbuild.gradleファイルに追加します。

    allprojects {
   repositories {
       jcenter()
       flatDir {
           dirs 'libs'
       }
   }
}

  2. tealium-5.x.x.aarを<PROJECT_ROOT>/<MODULE>/libsに追加します。

  3. Tealiumライブラリの依存関係をbuild.gradleファイルに追加します。

    dependencies {
    implementation(name:'tealium-5.5.5', ext:'aar')
}

初期化

以下のコードをアプリケーションのサブクラス内のonCreateメソッドに追加します。

Tealium.Config tealConfig = Tealium.Config.create(application,
                                                  "ACCOUNT",
                                                  "PROFILE",
                                                  "ENVIRONMENT");
tealConfig.setDataSourceId("DATASOURCEKEY");
tealConfig.setForceOverrideLogLevel("prod");
Tealium.createInstance("INSTANCE", tealConfig);
パラメータ 詳細
application アプリケーション。
account アカウント名。
profile プロファイル名。
environment “qa”または”prod”。
datasource （任意）UDHのデータソースキー。
instance 一意のインスタンス名（マルチインスタンスに対応）。

Collectが有効になっている場合には、
バージョン5.5.1以降 - Tealium.Configオブジェクトで提供されるプロファイルにデータが送信されます
バージョン5.5.0以前 - データがデフォルトのプロファイル（”main”）に送信されます

ビューのトラッキング

スクリーンビューは、スクリーン名と（任意で）コンテクスチュアルビューデータという、2つのパラメータを使用してtrackView()を呼び出すことでトラッキングできます。

スクリーン名の値はイベントデータ内でscreen_title変数として設定されています。

Map<String, Object> data = new HashMap<>(1);
data.put("someKey", "someValue");
Tealium.getInstance("INSTANCE").trackView("screenName", data);

イベントのトラッキング

非ビューイベントは、イベント名と（任意で）コンテクスチュアルビューデータという、2つのパラメータを使用してtrackEvent()を呼び出すことでトラッキングできます。

イベント名はイベントデータ内でtealium_event変数として設定されています。

Map<String, Object> data = new HashMap<>(1);
data.put("someKey", "someValue");
Tealium.getInstance("INSTANCE_NAME").trackEvent("someEvent", data);

永続性データ

永続性の値はデバイスでSharedPreferencesとして保存され、各イベントにマージされ、アプリを終了し、再度起動しても保持されます。

SharedPreferences sp = instance.getDataSources().getPersistentDataSources();
sp.edit().putInt("init_count", sp.getInt("init_count", 0) + 1).commit();

揮発性データ

揮発性の値は各イベントにマージされますが、アプリを終了して再起動すると破棄されます。

instance.getDataSources().getVolatileDataSources()
                .put("init_time", System.currentTimeMillis());

ログレベル

ログレベルはリモート公開設定により管理され、初期化の際に設定された環境から推測されます。

  • dev - activity、info、warnings、errors
  • qa - info、warnings、errors
  • prod - errors

バージョン5.3.1以降、setForceOverrideLogLevel()メソッドを使用してログレベルをローカルでオーバーライドできるようになっています。

// options: dev, qa, prod, silent
tealConfig.setForceOverrideLogLevel("dev");

Cookie

モバイルプロファイルでTag Managementを有効にしている場合、以下のコードを追加するとCookieの管理を有効にすることができます。 これは、初期化ステートメントの後に配置する必要があります。

config.getEventListeners().add(createCookieEnablerListener());
private static WebViewCreatedListener createCookieEnablerListener() {
    return new WebViewCreatedListener() {
        @Override
        public void onWebViewCreated(WebView webView) {
            final CookieManager mgr = CookieManager.getInstance();
            // Accept all cookies
            mgr.setAcceptCookie(true);
            if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.LOLLIPOP) {
                mgr.setAcceptThirdPartyCookies(webView, true);
            }
            if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.HONEYCOMB_MR1) {
                CookieManager.setAcceptFileSchemeCookies(true);
            }
            Log.d(TAG, "WebView " + webView + " created and cookies enabled.");
        }
    };
}

このコードにより、Tag Management WebViewがファーストパーティ、サードパーティ、ファイルスキームのCookieを受け入れることができます（[詳細](https://developer.android.com/reference/android/webkit/CookieManager.html#acceptCookie()）。

バージョン5.1.0以降では、CookieManagerがデフォルトで有効になっています。
バージョン5.0.4以前では、CookieManagerがデフォルトで無効になっています。
Tag Managementを使用する場合はCookieManagerを有効にする必要があります。

マルチプロセスアプリ

Android 9以降とマルチプロセスアプリケーション

Android 9では、マルチプロセスアプリケーションにおけるWebViewデータディレクトリに関する挙動が変更されました。 この変更により、複数のWebViewがマルチプロセスで1つのデータディレクトリを共有することができなくなりました。 通常は、1つのWebViewを共通して使用するActivityはいずれも、同一のプロセスで実行されます。 2つ以上のプロセスにおいてより多くのWebViewのインスタンスを実装し、TealiumのTag Managementソリューションを使用する場合は、アプリがクラッシュしないよう、Tealiumのインスタンスを作成する前にsetDataDirectorySuffix()を呼び出すことを推奨しています。

TealiumのTag ManagementのCookieとウェブデータにアクセスする必要がある場合は、CookieManagerのgetCookie()およびsetCookie()メソッドを使用して、プロセス間でコピーします。

その他の読み物：『プロセスで分けられたウェブベース データ ディレクトリ』
