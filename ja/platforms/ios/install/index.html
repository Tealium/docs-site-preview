IOS

インストール

Tealium for iOS SDKのコードを取得し、インストールします。

要件

  • Xcode 7以上
  • iOS 8.1以上との互換性

iQ Tag Managementと併用する場合：

  • iQのモバイルプロファイル（「モバイルプロファイルの作成」を参照してください）

Universal Data Hubと併用する場合：

  • Tealium Universal Data Hubアカウント

サンプルとリファレンス

iOSレポジトリ内で提供されているサンプルアプリを1つ以上ダウンロードすることを推奨しています。 これは当社のライブラリ、トラッキングメソッド、ベストプラクティスの実装について把握するのに役立ちます。

  • iOS サンプルアプリ
  • APIリファレンス

このほか、残りのアプリからTealiumの実装を抽出するためのヘルパークラスの使用も推奨しています。 これはトラッキングコールの初期化と実行の一元的なエントリポイントとなるもので、 このアプローチにより、1つひとつのViewControllerまたはヘッダー/メソッドファイルではなく、ヘルパーファイル内でコードをアップデートできるようになります。

  • サンプルヘルパークラス - Objective-C
  • サンプルヘルパークラス - Swift

インストール

CocoaPods

TealiumIOSフレームワークのインストールには、CocoaPodsの使用を推奨しています。これを行うには、Podfileに以下の依存関係を追加します。

pod "TealiumIOS"

ファイルを保存し、プロジェクトディレクトリで以下を実行します。

pod install

作成された.xcworkspaceファイルを使用して、アプリのビルドを継続します。 .xcworkspaceを使用しない場合、アプリを正しくビルドできません。

Carthage

ご希望の場合は、TealiumフレームワークのインストールにCarthageを使用することも可能です。

  1. Cartfileに以下を追加します。

    binary "https://tags.tiqcdn.com/dle/tealiummobile/tealium-ios-carthage/tealium-carthage.json"

  2. 以下のコマンドを使って、フレームワークのダウンロードと抽出を行います。

    carthage update --platform ios

  3. 必要なフレームワークがビルド時に自動的にコピーされるよう、XcodeプロジェクトにCarthageが適切に統合されているかを確認します。 

  4. App Storeに提出する前に、必要に応じてシミュレーターのサポートを削除します。 GitHubで提供されている”DeviceOnly”ビルドを使用してこの手順をスキップすることもできます。”DeviceOnly”ビルドはCarthage使用時にもダウンロードされます。

手動

以下の手順に従い、Tealiumを手動でプロジェクトに追加することもできます。

  1. TealiumIOS.frameworkをプロジェクトに追加します。
  2. 表示されるダイアログボックスで、フレームワークをプロジェクトにコピーします。
  3. [Target]で、[General] > [Embedded Binaries]に移動し、TealiumIOS.frameworkを追加します。

    Swiftプロジェクトの場合は、[Build Settings] > [Swift Compiler] > [Objective-C Bridging Header]オプションに移動し、TealiumIOSBridgingHeader.hファイルをインクルードします。

初期化

Tealiumインスタンスには、トラッキング開始前に以下のパラメータを構成する必要があります。

パラメータ 詳細
account アカウント名。
profile プロファイル名。
environment “qa”または”prod”。
datasource （任意）UDHのデータソースキー。
instance 一意のインスタンス名（マルチインスタンスに対応）。

アプリケーションデリゲートまたはヘルパークラスのセットアップメソッド内で、以下の初期化コードを使用します。  

    // set your account, profile, and environment
let tealConfig = TEALConfiguration.init(account: "ACCOUNT",
                                        profile: "PROFILE",
                                    environment: "prod",
                                     datasource: "DATASOURCE")

// Initialize with a unique key for this instance
guard let tealium = Tealium.newInstanceForKey("INSTANCE", configuration: tealConfig) else {
     // Any additional failure response here           
     return
}
    // import area
@import TealiumIOS;

// Set your account, profile, and environment
TEALConfiguration *tealConfig = [TEALConfiguration configurationWithAccount:@"YOUR_ACCOUNT"
                                                                    profile:@"YOUR_PROFILE"
                                                                environment:@"prod"
                                                                 datasource:@"DATASOURCE"];

// Initialize with a unique key for this instance
Tealium *tealium = [Tealium newInstanceForKey:@"INSTANCE" configuration:tealConfig]; 

    ログレベルについては、デフォルトではiQアカウントの公開設定が適用されます。 この設定をオーバーライドするには、TEALConfigurationインスタンスでlogLevelプロパティを設定します。

      // Set the log verbosity to errors only
tealConfig.logLevel = .prod
      // Set the log verbosity to maximum
[tealConfig setLogLevel: TEALLogLevelDev];

      ビューのトラッキング

      スクリーンビューのトラッキングには、trackViewWithTitle()を使用します。 これはどのView ControllerのviewDidAppearメソッドでも実行可能です。 このメソッドは、スクリーン名と（任意で）コンテクスチュアルビューデータという、2つのパラメータを受け取ります。 

      スクリーン名の値は、イベント属性screen_title内に設定されます。

        // Retrieve instance
let tealium = Tealium.instance(forKey: "INSTANCE")

// Basic
tealium?.trackViewWithTitle("screenName", dataSources: nil)

// With optional data
let optionalData = ["someKey": "someValue"]
tealium?.trackView(withTitle: "screenName", dataSources: optionalData)

// With optional array data
let optionalData = ["someKey": ["arrayValue1","arrayValue2"]]
tealium?.trackView(withTitle: "screenName", dataSources: optionalData)
        // Retrieve instance
Tealium *tealium = [Tealium instanceForKey:@"INSTANCE"];

// Basic
[tealium trackViewWithTitle:@"screenName" dataSources:nil];

// With optional data
NSDictionary *optionalData = @{@"someKey" : @"someValue"};
[tealium trackViewWithTitle:@"screenName" dataSources: optionalData];

// With optional array data
NSDictionary *optionalData = @{@"someKey" : @[@"arrayValue1", "arrayValue2"]};
[tealium trackViewWithTitle:@"screenName" dataSources: optionalData];

        イベントのトラッキング

        非ビューアクティビティのトラッキングには、trackEventWithTitle()を使用します。 このメソッドは、イベント名と（任意で）コンテクスチュアルイベントデータという、2つのパラメータを受け取ります。

        イベント名の値は、tealium_eventというイベント属性に入力されます。

          // Retrieve instance
let tealium = Tealium.instance(forKey: "INSTANCE")

// Basic
tealium?.trackEvent(withTitle: "someEvent", dataSources: nil)

// With optional data
let optionalData = ["someKey": "someValue"]
tealium?.trackEvent(withTitle: "someEvent", dataSources: optionalData)

// With optional array data
let optionalData = ["someKey": ["arrayValue1", "arrayValue2"]]
tealium?.trackEvent(withTitle: "someEvent", dataSources: optionalData)
          // Retrieve instance
Tealium *tealium = [Tealium instanceForKey:@"INSTANCE"];

// Basic
[tealium trackEventWithTitle:@"someEvent" dataSources:nil];

// With optional data
NSDictionary *optionalData = @{@"someKey" : @"someValue"};
[tealium trackEventWithTitle:@"someEvent" dataSources:optionalData];

// With optional array data
NSDictionary *optionalData = @{@"someKey" : @[@"arrayValue1", @"arrayValue2"]};
[tealium trackEventWithTitle:@"someEvent" dataSources:optionalData];

          永続性データと揮発性データ

          上記のtrackViewやtrackEvent関数経由で送信できる一時的なイベントデータ以外に、各トラッキングコールの共通データを設定する関数もあります。

            永続性の値と揮発性の値は、デバイスに保存されています。 永続性オブジェクトは各イベントにマージされ、アプリを終了し、再度起動しても保持されます。揮発性オブジェクトは、アプリを終了して再起動すると破棄されます。

            //persistent data
tealium.persistentData()?.add(data: ["persistent_key": "persistent_value"])
let persistentData = tealium.persistentData()

//volatile data
tealium.volatileData()?.add(data: ["volatile_key": "volatile_value"])
let volatileData = tealium.volatileData()

            永続性の値と揮発性の値は、NSDictionaryオブジェクトとしてデバイスに保存されています。 永続性オブジェクトは各イベントにマージされ、アプリを終了し、再度起動しても保持されます。揮発性オブジェクトは、アプリを終了して再起動すると破棄されます。

            //persistent data
[self.tealium addPersistentDataSources:@{@"persistent_key": @"persistent_value"}];
NSDictionary *persistentDataSources = [self.tealium persistentDataSourcesCopy];

//volatile data
[self.tealium addVolatileDataSources:@{@"volatile_key": @"volatile_value"}];
NSDictionary *volatileDataSources = [self.tealium volatileDataSourcesCopy];

            App Storeへの提出

            モバイルライブラリには、シミュレーターおよびデバイスで実行されるスライスが含まれます。 Appleでは、App Storeへの提出時にシミュレーターのアーキテクチャが残っていてはならないとしています。  そのため、アーカイブを作成する際に、以下の2つのスクリプトのいずれかを実行し、シミュレーターを削除する必要があります。

            • Daniel Kennett氏のStrip Framework Script
            • Realmのstrip-frameworks.sh

            このほか、シミュレーターのスライスを含まないDevicesOnly ビルドを使用することもできます。
