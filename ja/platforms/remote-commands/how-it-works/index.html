REMOTE COMMANDS

仕組み

リモートコマンドは、iQ Tag Management内のタグを使用してネイティブアプリ内のコードをトリガーするための、クライアントサイドのソリューションです。

リモートコマンドにより、ネイティブアプリのコードが持つ力と、iQ Tag Managementの利便性とタグ管理を統合できます。 リモートコマンドでは、アプリ内でJavaScriptタグによってメソッドをトリガーする方法として、カスタムURLスキームが使用されます。 レンダリングされていないウェブビュー内のタグによりカスタムリクエストが送信され、それをアプリが受信します。 これらのリクエストは、カスタムネイティブコードまたはベンダーSDKをトリガーするリモートコマンドハンドラーによって処理されます。

リモートコマンドは次の2通りの方法で使用できます。

  • ベンダー統合 - ベンダーAPIを実装するネイティブリモートコマンドモジュールと、iQ Tag Management内の対応するベンダータグを使用しているベンダーSDK用に、事前にビルドされたソリューション （リモートコマンド統合のリストを参照してください）。
  • カスタムコマンド - ネイティブコードで定義され、iQ Tag ManagementでTagBridgeのカスタムコマンドタグにより構成された、自分のカスタムコマンド。

コンポーネント

リモートコマンドはトラッキングされたイベントと似た構造をしており、コマンド名とデータのペイロードがあります。

  • コマンド：コマンド名、または”コマンドID”。アプリ内のネイティブコードとTealium iQ内のタグに登録されています。
  • ペイロード：タグからネイティブアプリに渡されるデータ。データマッピングとして構成され、アプリ内でハンドラーのレスポンスコールバックのrequestPayloadオブジェクトとして受信されます。

リモートコマンドはビルド時にネイティブアプリで定義されている必要があります。 リモートコマンドタグでは、デバイス上で定義済みのコードのみ実行可能です。

ネイティブコード

ネイティブコードでは、addRemoteCommandIDというヘルパー関数が、リモートコマンド用のハンドラーを登録します。 コールバック関数responseBlockは、response.requestPayload内のペイロードにアクセスできます。 ここで参照されるペイロード変数には、iQ Tag Managementで構成され、対応するデータレイヤー変数があるはずです。

TealiumHelper.shared.tealium.addRemoteCommandID(
    "myRemoteCommand",
    description: "",
    targetQueue: DispatchQueue.main,
    responseBlock: { response in
        guard let myVariable = response.requestPayload["myVariable"] as? String else {
            return
        }
    }
)

TagBridgeカスタムコマンドタグ

iQ Tag Managementでは、リモートコマンドはTagBridgeカスタムコマンドタグのインスタンスとして、以下の設定でセットアップされます。

  • コマンドID
    コマンド名（ネイティブメソッドaddRemoteCommandIDで使用されているのと同じ名前）。
  • タグ配信ルール
    リモートコマンドをトリガーするタイミングを定めたルール。
  • マッピングされた変数
    リモートコマンドに含めるデータ。 マッピングされたデータレイヤー変数は、ネイティブコードコールバックのresponse.requstPayloadオブジェクトで利用できます。

リモートコマンドタグの例：

トラッキングされたイベント：

tealium?.track(
    title: "My Screen",
    data: ["tealium_event": "my_event", "my_variable": "my_value"]
);

タグのセットアップ：

  • コマンドID
    myRemoteCommand
  • タグ配信ルール
    IF tealium_event EQUALS my_event
  • マッピングされた変数
    my_variable -> myVariable

ベンダー統合

ベンダー統合は、事前にビルドされたリモートコマンドモジュールであり、ベンダーSDKをインストールしたり、ベンダーAPI用のネイティブコードハンドラーを定義したりします。

リモートコマンド統合の実装には、以下が必要です。

  • ベンダーリモートコマンドモジュール
    アプリのビルドスクリプトで、ベンダーSDKをリモートコマンドモジュールに置換します。 リモートコマンドモジュールにより、ベンダーSDKのインストールとセットアップが処理されます。
  • ベンダーリモートコマンドタグ
    iQ Tag Managementで、ベンダー統合用のリモートコマンドタグを追加します。 目的のベンダーメソッドを実装するには、タグ設定とデータマッピングを構成します。

ネイティブコード

ベンダー統合用のモジュールを、お使いのプラットフォーム用のビルドスクリプトに追加することもできます。 モジュールはTealiumVendorXYZの形式で命名されます。例：TealiumBraze。

インストールするには、ベンダーの依存関係をTealiumのリモートコマンドの依存関係に置換します。 例：

// Remove this line
pod "VendorXYZ-iOS-SDK"

// Add this line
pod "TealiumVendorXYZ"

リモートコマンドタグ

iQ Tag Managementでは、ベンダー統合用のリモートコマンドタグが提供されています。 リモートコマンドタグは、通常のウェブサイトのタグのように使用します。 各リモートコマンドタグは、ベンダーのSDKメソッド用のイベントマッピングを提供します。イベントマッピングは、アプリのモジュールでリモートコマンドとして定義されています。
