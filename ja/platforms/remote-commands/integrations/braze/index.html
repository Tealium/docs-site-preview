REMOTE COMMANDS/INTEGRATIONS

リモートコマンド：Braze（ベータ）

AndroidおよびSwift/iOSにおけるBrazeのTealiumリモートコマンド統合です。

要件

  • Braze APIキー
  • AndroidまたはSwift/iOS用のTealiumライブラリ（アプリ内）
  • Brazeリモートコマンドタグ（iQ Tag Management）

仕組み

Braze統合では、ネイティブBraze SDK、Brazeメソッドをラップするリモートコマンドモジュール、およびイベントトラッキングをネイティブBrazeコールに変換するBrazeリモートコマンドタグを使用します。 このソリューションでは、タグ管理の利便性を活用して、アプリにベンダー固有のコードを追加することなくネイティブのBraze実装を構成します。

Braze SDKを個別にインストールする必要はありません。 Brazeリモートコマンドモジュールをアプリビルドに追加すると、必要なBrazeライブラリが自動的にインストールされ、ビルドされます。 以前にBrazeをアプリに追加した場合は、それを削除してかまいません。

インストール

依存関係マネージャ

以下の依存関係マネージャではインストールオプションを使用できます。 これは、推奨のインストール方法です。

    次の依存関係をPodfileに追加します。

    pod "TealiumBraze"

    さらに、pod "Appboy-iOS-SDK"という行がPodfileに存在する場合は、その行を削除します。

    TealiumBrazeには既にtealium-swiftへの依存関係があるため、Podfileにtealium-swiftがあった場合は、それを削除する必要があります。

    次の依存関係をCartfileに追加します。

    github "tealium/tealium-ios-braze-remote-command"

    さらに、github "Appboy/Appboy-iOS-SDK"という行がCartfileに存在する場合は、その行を削除します。

    TealiumBrazeには既に`tealium-swift`への依存関係があるため、Cartfileに`tealium-swift`があった場合は、それを削除する必要があります。 バージョン1.6.5以降のTealium for Swift SDKを使用している場合は、**TealiumDelegate**モジュールもインストールに含める必要があります。

    build.gradleファイルに以下への参照を追加します。

    dependencies {
    implementation 'com.tealium.remotecommands:braze:1.0.0'
}

    iOS向けの手動による手順

    Objective-CまたはSwiftプロジェクトでは、Brazeリモートコマンドは次のレポジトリを使用して手動でインストールできます。

    • Tealium iOS Brazeリモートコマンド

    このリモートコマンドファイルを独自のプロジェクトに含めるには、このレポジトリをクローンし、プロジェクトのSourcesフォルダにファイルをドラッグします。

    この手順では、Tealium SwiftとBrazeが既にインストールされていることを想定しています。

    Tealium SDKを初期化する際には、完了ハンドラーを以下のようにします。

    tealium = Tealium(config: config) { responses in
  guard let remoteCommands = self.tealium?.remoteCommands() else {
      return
  }
  let brazeCommandRunner = BrazeCommandRunner()
  let brazeCommand = BrazeCommand(brazeCommandRunner: brazeCommandRunner)
  let brazeRemoteCommand = brazeCommand.remoteCommand()
  remoteCommands.add(brazeRemoteCommand)
}

    手動インストールオプションは、以下のプラットフォームで使用できます。

      まず、レポジトリをクローンして、プロジェクトのObjective-Cフォルダにファイルをドラッグします。

      次に、TealiumHelper.mに、Swiftコードをインポートするための次の行を追加します。

      #import "TealiumRemoteCommandObjcApp-Swift.h"

      Objective-CにSwiftをインポートする方法の詳細については、以下を参照してください。
      Apple Developerドキュメント：Importing Swift Into Objective-C

      その後、リモートコマンドを初期化します。

      BrazeCommandRunner *brazeCommandRunner = [[BrazeCommandRunner alloc] init];
BrazeCommand *brazeCommand = [[BrazeCommand alloc] initWithBrazeCommandRunner:brazeCommandRunner];
TEALRemoteCommandResponseBlock brazeRemoteCommand = [brazeCommand remoteCommand];
[[Tealium instanceForKey:@"MY_INSTANCE"] addRemoteCommandID:@"braze"
                                    description:@"Braze Remote Command"
                                    targetQueue:dispatch_get_main_queue()
                                    responseBlock:brazeRemoteCommand];

      TealiumHelper.swiftに、リモートコマンドを次のように追加します。

      let brazeCommandRunner = BrazeCommandRunner()
let brazeCommand = BrazeCommand(brazeCommandRunner: brazeCommandRunner)
let brazeRemoteCommand = brazeCommand.remoteCommand()
tealium.addRemoteCommandID("braze",
                           description: nil,
                           targetQueue: DispatchQueue.main,
                           responseBlock: brazeRemoteCommand)

      初期化

      すべてのTealiumライブラリでは、初期化時にBrazeリモートコマンドを登録する必要があります。

        Tealium.Config config = Tealium.Config.create(application, "your_account", "your_profile", "dev");
Tealium teal = Tealium.createInstance(TEALIUM_MAIN, config);

RemoteCommand braze = new BrazeRemoteCommand(application);
teal.addRemoteCommand(braze);

        このクラスはObjective-Cライブラリ用に設計されていますが、コード自体はSwiftで記述されています。 これが適切に機能するためには、Xcodeプロジェクトにブリッジヘッダーを用意する必要があります。

        let config = TEALConfiguration(account: "your_account",
                               profile: "your_profile",
                               environment: "dev",
                               datasource: "abc123")
let teal = Tealium.newInstance(forKey: "teal", configuration: config)

let brazeCommandRunner = BrazeCommandRunner()
let brazeCommand = BrazeCommand(brazeCommandRunner: brazeCommandRunner)
let brazeRemoteCommand = brazeCommand.remoteCommand()
teal.addRemoteCommandID("braze", description: nil, targetQueue: DispatchQueue.main, responseBlock: brazeRemoteCommand)

        このクラスは、iOS向けのTealiumのSwiftライブラリと一緒に使用するように設計されています（他のプラットフォームでは動作しません）。

                           datasource: "abc123",
                           optionalData: nil)

        return
  }

        サポートされているメソッド

        以下のBrazeメソッドは、以下のTealiumコマンドを使用し、Brazeリモートコマンドタグのデータマッピングを使用してトリガーできます。

        リモートコマンド Brazeメソッド
        initialize startWithApiKey()
        useridentifier changeUser()
        useralias addAlias()
        userattribute ABKUser()
        facebookuser facebookUser()
        twitteruser twitterUser()
        setcustomattribute setCustomAttributeWithKey()
        unsetcustomattribute unsetCustomAttributeWithKey()
        setcustomarrayattribute setCustomAttributeArrayWithKey()
        appendcustomarrayattribute addToCustomAttributeArrayWithKey()
        removecustomarrayattribute removeFromCustomAttributeArrayWithKey()
        incrementcustomattribute incrementCustomUserAttribute()
        emailnotification setEmailNotificationSubscriptionType()
        pushnotification setPushNotificationSubscriptionType()
        logcustomevent logCustomEvent()
        logpurchase logPurchase()

        Braze SDKはTealium SDKとともにインストールされるので、対応するタグ構成があれば任意のネイティブBraze機能をトリガーできます。

        SDKのセットアップ

        初期化

        Braze SDKは起動時に自動的に初期化されます。 Braze APIキーはタグ構成で設定されます。

        リモートコマンド Brazeメソッド
        initialize startWithApiKey()

        起動オプションの設定には、以下のマッピングを使用できます。

        パラメータ 詳細 型
        session_timeout セッションタイムアウト（秒単位。デフォルト：30秒） 整数（秒）
        disable_location 位置データの自動収集を無効にします。 整数（1または0）
        trigger_interval_seconds アプリ内メッセージがトリガーされる最小間隔をオーバーライドします（デフォルト：30秒） 整数（秒）
        • Braze Developer Guide: SDKの初期セットアップ
