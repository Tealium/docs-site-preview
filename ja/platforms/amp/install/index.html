AMP

インストール

AMPサイトにTealium Collectをインストールする方法です。

要件

  • Tealium Universal Data Hubアカウント

仕組み

<amp-analytics>要素は、AMPドキュメントから分析データを取得するAMP拡張コンポーネントです。 ページの<head>セクション内に以下のスクリプトを追加します。

<script async custom-element="amp-analytics" src="https://cdn.ampproject.org/v0/amp-analytics-0.1.js"></script>

Tealium Collectは、AMP分析の使用のためのビルトイン構成として利用可能です。 Tealium CollectのベースインストールをAMPページに追加するには、typeプロパティをtealiumcollectに設定した状態で<amp-analytics>要素をページの<body>に追加してから、初期化パラメータとともにvarsブロックを追加します。

<amp-analytics type="tealiumcollect">
<script type="application/json">
{
  "vars": {
    "account": "ACCOUNT",
    "profile": "PROFILE",
    "datasource": "DATASOURCE"
  }
}     
</script>
</amp-analytics>
パラメータ 詳細
account Tealiumアカウントの名前。
profile Tealiumプロファイルの名前。
datasource （任意）UDHのデータソースキー。

ページトラッキング

ベースインストールにより、ユーザーがAMPサイトの画面の表示や変更をするたびにトラッキングコールが実行されます。 このトラッキングに使用されるデータレイヤーをカスタマイズするには、変数のキーと値のペアとともにextraUrlParamsブロックを追加します。

<amp-analytics type="tealiumcollect">
<script type="application/json">
{
  "vars": {
    "account": "ACCOUNT",
    "profile": "PROFILE",
    "datasource": "DATASOURCE"
  },
  "extraUrlParams": {
    "example_param1": "abc",
    "example_param2": 123
  }
}     
</script>
</amp-analytics>

extraUrlParams（amp-analytics）の詳細については、リンク先を参照してください。

イベントトラッキング

構成オブジェクトのtriggersでは、トラッキングするイベントを定義します。 トリガーは、トリガー名と構成を表すキーと値のペアとして定義されます。 トリガー名には任意の英数字の文字列を使用できます。構成ではイベントのターゲットを決定します。

"triggers": {
  "trigger_name": {
    "on": "VALUE",
    "selector": "VALUE",
    "request": "VALUE",
    "vars": {
      "tealium_event": "VALUE"
    }
    "extraUrlParams": {
      "extra_var": "${extraVar}"
    }
  }
}

AMPでは以下のトリガーパラメータがサポートされています。

パラメータ 詳細
on （必須）監視するイベント。 有効な値はclick、scroll、timer、visible。
request （必須）送信するリクエストのタイプを指定します。
vars 最上位の構成で定義されたvarsをオーバーライドしたり、このトリガー特有の変数を指定したりするのに使用されるキーと値のペアを含むオブジェクト。
extraUrlParams 動的なイベント変数のキーと値のペアを含むオブジェクト。
selector （onがclickに設定されている場合は必須）トラッキングする要素を絞り込むCSSセレクター。
scrollSpec （onがscrollに設定されている場合は必須）ページがスクロールされたときに、どのような条件でscrollイベントが発動されるかを制御します。
timerSpec （onがtimerに設定されている場合は必須）timerイベントがいつ発動されるかを制御します。

amp-analyticsトリガーの詳細については、リンク先を参照してください。

動的なイベント変数

visibleとclickのイベントタイプ（onプロパティで使用）では、data-vars-*の形式を使用することによって、HTML5の要素の属性を使用して動的なトラッキング変数を渡すことができます。 この方法を使って渡された変数はキャメルケースを使った名前に変換されます。 たとえば、data-vars-link-name="register"というプロパティは、変数${linkName}となり、トリガーのextraUrlParamsブロック内で参照可能です。

例：

"triggers": {
  "custom_click": {
    "on": "click",
    "selector": "#the-button",
    "request": "event",
    "vars": {
      "tealium_event": "custom_click"
    }
    "extraUrlParams": {
      "link_name": "${linkName}"
    }
  }
}
<span id="the-button" data-vars-link-name="register">
Click here to register
</span>

検証

ウェブページがAMPのHTMLの仕様を満たしているかどうかを検証するには、AMP Validatorを使用します。 バリデーターをオンにするには、AMPページのURLの末尾に#development=1を追加します。 AMPのバリデーションエラーは、ブラウザのコンソールに表示されます。

ホストされたJSON

Tealium Collect for AMPを（ページに直接埋め込まれたJSONではなく）ホストされたJSONを使用してインストールする場合は、config属性を使用してホストされた場所からAMP構成を読み込みます。

<amp-analytics type="tealiumcollect" config="https://tags.tiqcdn.com/dle/ACCOUNT/PROFILE/FILE.json" />

AMPサイトで使用するためにリモートJSON構成ファイルを提供する方法の詳細については、リンク先を参照してください。

ページ全体の例

以下は完全なAMPページの例です。 この構成では、標準的なページビューデータがTealiumへ送信され、ボタンクリックがトラッキングされます。

<!doctype html>
<html amp lang="en">
  <head>
    <meta charset="utf-8">
    <script async src="https://cdn.ampproject.org/v0.js"></script>
    <script async custom-element="amp-analytics" src="https://cdn.ampproject.org/v0/amp-analytics-0.1.js"></script>
    <style amp-boilerplate>body{-webkit-animation:-amp-start 8s steps(1,end) 0s 1 normal both;-moz-animation:-amp-start 8s steps(1,end) 0s 1 normal both;-ms-animation:-amp-start 8s steps(1,end) 0s 1 normal both;animation:-amp-start 8s steps(1,end) 0s 1 normal both}@-webkit-keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}@-moz-keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}@-ms-keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}@-o-keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}@keyframes -amp-start{from{visibility:hidden}to{visibility:visible}}</style><noscript><style amp-boilerplate>body{-webkit-animation:none;-moz-animation:none;-ms-animation:none;animation:none}</style></noscript>
    <title>Hello, AMPs</title>
    <link rel="canonical" href="http://example.ampproject.org/article-metadata.html">
    <meta name="viewport" content="width=device-width,minimum-scale=1,initial-scale=1">
  </head>
  <body>
    <amp-analytics type="tealiumcollect">
    <script type="application/json">
    {
      "vars": {
        "account": "ACCOUNT",
        "profile": "PROFILE",
        "datasource": "DATASOURCE"
      },
      "triggers": {
        "custom_click": {
          "on": "click",
          "selector": "#the-button",
          "request": "event",
          "vars": {
            "tealium_event": "custom_click"
          },
          "extraUrlParams": {
            "link_name": "${linkName}"
          }
        }
      },
      "extraUrlParams": {
        "site_section": "Demos"
      }
    }
    </script>
    </amp-analytics>

    <h1 id="header">Welcome to the mobile web.</h1>
    <p>Testing AMP Analytics "tealiumcollect" component.</p>
    <div>
      <button type="button" id="the-button" data-vars-link-name="track">Track</button>
    </div>
  </body>
</html>
