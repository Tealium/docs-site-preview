HTTP API

エンドポイント仕様

イベントにHTTP APIを使用してUniversal Data Hubにデータを送信する方法を説明します。この記事では、HTTPリクエストを送信するアプリケーションで使用できるダイレクトHTTPエンドポイントの仕様を紹介します。

ルートURL

Tealium Collectは次のルートURLにあります。 

https://collect.tealiumiq.com/event

標準パラメータ

Tealium Collectは、各リクエストで次の標準パラメータをサポートしています。

パラメータ 詳細
tealium_account Tealiumアカウントの名前
tealium_profile Tealiumプロファイルの名前（デフォルトは”main”）
tealium_datasource （任意）UDHのデータソースキー（詳細は、「データソース」を参照してください）
tealium_event （推奨）トラッキング目的のイベントの名前
tealium_visitor_id （AudienceStreamに必須）イベントに関連付けられた訪問者の匿名の一意識別子（後述の訪問者IDに関するセクションを参照）
tealium_trace_id （任意）トレースと一緒に使用

リクエストの形式は、使用されているHTTPメソッドによって異なります。 以降の例では、プレースホルダーの値は、波括弧を使った{VALUE}形式で表現しています。

カスタムイベントパラメータ

追加のイベント属性は、トラッキングのニーズに応じてカスタムパラメータとして送信されます。 このようなパラメータはUDHのイベント属性として定義します。

訪問者ID

AudienceStreamを使用している場合は、既知の訪問者ID属性（セカンダリID）を各トラッキングコール（例：email_addressやlogin_id）と一緒に渡す必要があります。 イベントに訪問者IDが含まれていない場合は、イベントごとに新しい訪問と訪問者が生成されるので、スティッチ（縫い合わせ）は不可能です。 これが特に重要なのは、匿名ユーザーを追跡しているウェブプラットフォームでCollect APIを使用している場合です。

また、EventStreamを使用していてベンダーコネクタにIDを指定する必要がある場合も訪問者IDが必要です。

適切な訪問者IDを作成する条件の詳細は、「訪問者ID属性」を参照してください。

プライマリ訪問者ID

tealium_visitor_id属性は、イベントと訪問の間に訪問者を関連付けるためにAudienceStreamで使用される匿名の識別子です。Collect APIの使用方法に応じて、次のいずれかのアプローチを使用できます。

  • 既知の訪問者（AudienceStreamの場合は必須、EventStreamの場合は推奨）
    既知の訪問者のイベントをトラッキングする場合（email_addressなどの訪問者ID属性に値が存在する場合）、既知の訪問者ID属性とその値のハッシュをtealium_visitor_idパラメータに含めます。

    Example:

    {
    "tealium_account"    : "my_account",
    "tealium_profile"    : "main",
    "tealium_event"      : "user_login",
    "email_address"      : "user@example.com",
    // MD5 Hash of user@example.com
    "tealium_visitor_id" : "b58996c504c5638798eb6b511e6f49af"
}

  • 匿名の訪問者（AudienceStreamの場合は必須）
    ウェブサイトの匿名の訪問者のイベントを追跡する場合（既知の識別子なし）、tealium_visitor_idの値はutag_main_v_idのCookieの値と一致させる必要があります。 詳細は、utag.jsのビルトイン変数に関するページを参照してください。

POSTメソッド

このPOSTメソッドがサポートするJSONペイロードでは、リクエストヘッダーをContent-Type: application/jsonに設定し、ペイロードを有効なJSON文字列形式に設定する必要があります。 

user_loginイベントの例：

{
    "tealium_account"   : "{ACCOUNT}",
    "tealium_profile"   : "{PROFILE}",
    "tealium_event"     : "user_login",
    "email_address"     : "user@example.com"
}

curlの使用例：

curl -X POST -H 'Content-type: application/json' \
--data '{"tealium_account":"{ACCOUNT}","tealium_profile":"{PROFILE}","tealium_event":"user_login","email_address":"user@example.com"}' \
https://collect.tealiumiq.com/event

GETメソッド

このGETメソッドは、エンドポイントのクエリ文字列にキーと値のペアを使用してデータを渡します。 このメソッドにより、1x1の透過GIFが返され、HTMLベースの実装との互換性が維持されます。

curlの使用例：

curl -i -X GET "https://collect.tealiumiq.com/event?tealium_account={ACCOUNT}&tealium_profile={PROFILE}&tealium_event=user_login&email_address=user@example.com" 

HTMLの例

Tealium Collectは、HTML内では<img>タグで参照できます。 ブラウザのキャッシュのため、キャッシュバスター（ランダムに生成された値を含む追加パラメータ）を含めることが重要です。

キャッシュバスターの例：

var cb=Math.random()*100000000000;

この変数は、Tealium Collectのピクセルのクエリ文字列に追加します。

<img height="1" width="1" style="display:none" src="//collect.tealiumiq.com/event?tealium_account={ACCOUNT}&tealium_profile={PROFILE}&tealium_event=user_login&email_address=user@example.com&cb=' + cb + '"/>

JavaScriptの例

Tealium Collectではクロスドメインリクエストがサポートされているため、ウェブサイトからTealiumのドメインにPOSTを送信できます。 レスポンスヘッダーにはAccess-Control-Allow-Origin: [http://your_domain.com](http://your_domain.com)が含まれます。

var event = {
    "tealium_account" : "ACCOUNT",
    "tealium_profile" : "PROFILE",
    "tealium_event"   : "user_login",
    "email_address"   : "user@example.com"
};
var xhr = new XMLHttpRequest();
xhr.open("POST", "https://collect.tealiumiq.com/event");
xhr.send(JSON.stringify(event));
